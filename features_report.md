## 3. Backend Requirements (Required APIs)

To make the application functional, the following backend services and endpoints are needed:

### Authentication
- `POST /api/auth/login`: Authenticate user.
- `POST /api/auth/logout`: Terminate session.
- `GET /api/auth/me`: Get current user profile.

### Contacts Management
- `GET /api/contacts`: Fetch all contacts (with search/pagination).
- `POST /api/contacts`: Create a new contact.
- `PATCH /api/contacts/:id`: Update contact details.
- `DELETE /api/contacts/:id`: Remove a contact.

### Messaging System
- `GET /api/conversations`: List all active conversations.
- `GET /api/conversations/:id/messages`: Fetch message history.
- `POST /api/messages/send`: Send a new message.
- `WS /ws/messages`: Real-time message updates (Socket.io/WebSockets).

### Call Management
- `GET /api/calls`: Fetch call history logs.
- `POST /api/calls/initiate`: Start a call (requires VOIP provider integration like Twilio).
- `WS /ws/calls`: Real-time call status updates.

### Analytics
- `GET /api/analytics/usage`: Data for charts (calls per day, message volume).

## 4. Integration Strategy

1.  **API Client:** Create a centralized `src/lib/api-client.ts` using `axios`.
2.  **Data Fetching:** Replace static arrays in pages with `useQuery` hooks from `@tanstack/react-query`.
3.  **State Management:** Use `useMutation` for handling form submissions (Add Contact, Send Message).
4.  **Environment Variables:** Configure `.env` for `VITE_API_URL`.

## 5. Recommended Backend Stack
- **Node.js (Express/FastAPI)** for the API layer.
- **PostgreSQL** for relational data (Contacts, Calls, Messages).
- **Redis** for real-time features and caching.
- **Twilio/SignalWire API** for actual VOIP/SMS functionality.
